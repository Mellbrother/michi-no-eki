# michi-no-eki

スタンプラリー形式で「道の駅」を巡るモバイルアプリの計画リポジトリです。ユーザーは現在地に基づき近隣の道の駅を探し、現地でスタンプを集めながら都道府県別の訪問状況を確認できます。

## 目的
- 地域観光の活性化と回遊促進を支援する。
- スタンプ収集や達成状況の可視化を通じて継続的な利用体験を提供する。
- 道の駅データを一元管理し、運営者向けの分析/連携を可能にする。

## 主要機能 (計画)
- **位置情報取得**: 現在地から近い道の駅を検索し、距離や移動ルートを提示。
- **スタンプ収集**: GPS 位置や QR コード入力をトリガーにスタンプを登録し、ポイントや達成状況を更新。
- **都道府県別一覧/達成率**: 都道府県ごとに登録されている道の駅を一覧表示し、スタンプ済み/未訪問を区別。達成率（制覇割合）を可視化。
- **プロフィール管理**: 表示名・アイコン・自己紹介編集、累積スタンプ数や達成率の確認。
- **ヘルスチェック/運用**: API のヘルスエンドポイント、ログ/モニタリング連携を前提とした運用を想定。

## 技術スタック (想定)
- **モバイル**: React Native (Expo)、TypeScript、Jest、ESLint。
- **バックエンド**: FastAPI、Python 3.11、SQLAlchemy、Pydantic、pytest、Ruff。
- **データベース**: PostgreSQL + PostGIS 拡張を検討（位置情報検索のため）。
- **CI/CD**: GitHub Actions で lint / test を自動実行し、将来的にビルド・デプロイを追加。
